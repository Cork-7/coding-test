### 성능 차이 분석
1. 반복문 (For loop) 방식
    - 이 방식은 숫자가 $n$개 있으면, 실제로 $n$번의 덧셈 과정을 거친다
    - 동작: start부터 end까지 하나하나 방문하며 + 연산을 수행
    - 시간 복잡도: $O(n)$
    - 특징: 숫자의 범위가 100이라면 100번 계산하지만, 1억이라면 1억 번의 연산 즉, 데이터의 양에 비례해서 시간이 늘어난다
2. 가우스 공식 (수학적 계산) 방식
   - 이 방식은 숫자가 몇 개든 상관없이, 단 한 번의 수식 계산으로 끝난다
   - 동작: (첫 항 + 끝 항) * 항의 개수 / 2 라는 공식을 바로 적용
   - 시간 복잡도: $O(1)$ (상수 시간)
   - 특징: 숫자가 10개든, 100억 개든 컴퓨터 입장에서는 단 한 번의 곱셈과 나눗셈, 데이터의 양과 상관없이 처리 속도가 항상 일정하고 매우 빠르다

---

### 문제 설명
- 두 정수 a, b가 주어졌을 때 a와 b 사이에 속한 모든 정수의 합을 리턴하는 함수, solution을 완성하세요.

- 예를 들어 a = 3, b = 5인 경우, 3 + 4 + 5 = 12이므로 12를 리턴합니다.

---

###  제한 사항
- a와 b가 같은 경우는 둘 중 아무 수나 리턴하세요.
- a와 b는 -10,000,000 이상 10,000,000 이하인 정수입니다.
- a와 b의 대소관계는 정해져있지 않습니다.

---

### 입출력 예
| a | b | return |
|:--|:--|:-------|
| 3 | 5 | 12     |
| 3 | 3 | 3      |
| 5 | 3 | 12     |
