# 서울에서 김서방 찾기

### 성능 요약
- 1번 방식
- 메모리 : 5.55MB
- 시간 : 77.7ms

---

- 2번 방식
- 메모리: 5.57MB
- 시간: 80.2ms

---

- 3번 방식
- 메모리 : 0.07MB
- 시간: 80.5ms

---

### 정리

1. for문 + + 연산 (가장 표준적)
> 성능 특징: 추가 객체 생성 없이 배열을 직접 훑기 때문에 메모리 효율이 매우 좋습니다.
>
> 차이점: "김서방은 " + i + "에 있다" 부분에서 내부적으로 새로운 String 객체가 생성되지만, 단 한 번만 실행되므로 성능 영향은 거의 없습니다.
>
> 한줄 요약: 가장 직관적이고 자원을 적게 쓰는 정석적인 방법.

2. Arrays.asList() (가장 간결함)
>성능 특징: 내부적으로 배열을 리스트로 감싸는 ArrayList 객체를 새로 생성해야 합니다.
> 
>차이점: for문을 직접 돌리는 것보다 아주 미세하게 메모리를 더 쓰지만, 코드가 매우 짧아집니다. (다만, 대용량 데이터에서는 객체 생성 비용이 발생할 수 있음)
> 
>한줄 요약: 생산성(코드 길이)은 최고지만, 아주 미세한 메모리 오버헤드가 있음.

3. StringBuilder (대규모 작업에 최적)
> 성능 특징: 새로운 문자열 객체를 계속 만들지 않고, 하나의 가변 버퍼에서 문자를 수정합니다.
>
> 차이점: 이 문제처럼 문장을 한 번만 만들 때는 1번 방식과 차이가 거의 없으나, 만약 수천 번 문자열을 합쳐야 하는 상황이라면 1, 2번보다 압도적으로 빠릅니다.
>
> 한줄 요약: 문자열 수정이 빈번할 때 최고의 성능을 발휘하는 '대용량 전용' 도구.

---

### 문제 설명

String형 배열 seoul의 element중 "Kim"의 위치 x를 찾아, "김서방은 x에 있다"는 String을 반환하는 함수, solution을 완성하세요.

seoul에 "Kim"은 오직 한 번만 나타나며 잘못된 값이 입력되는 경우는 없습니다.

---

### 제한 사항

- seoul은 길이 1 이상, 1000 이하인 배열입니다.
- seoul의 원소는 길이 1 이상, 20 이하인 문자열입니다.
- "Kim"은 반드시 seoul 안에 포함되어 있습니다.

---

### 입출력 예

| seoul          | return       |
|:---------------|:-------------|
| ["Jane","Kim"] | "김서방은 1에 있다" |
